JISコード

「JISコード」という用語は, しばしば「ISO-2022-JP」と混同して使用されることがありますが, ここで述べるJISコードとは, 「いわゆる半角カタカナの代表的な拡張３種類が加えられた, ASCII, JIS X 0201, JIS X 0208文字集合のISO-2022-JPエンコーディング」のことです。

本来, 「JISコード」とは, 日本工業規格が定めた文字セットおよびそれに対応する文字コードのことを指す用語です。JISコードの中でも, 特に日本語の漢字に関わるものは「JIS漢字コード」と呼ばれ, 「JIS X 0208」と「JIS X 0213」が特に有名です。後者は前者の上位集合であり, 前者は「第一・第二水準漢字」, 後者は「第一・第二・第三・第四水準漢字」で構成されています。JIS X 0208には1978年版・1983年版・1990年版・1997年版があり, JIS X 0213には2000年版・2004年版が存在します。JIS X 0208の初版を「旧JIS」, 1983年版を「新JIS」と呼ぶこともあります。

「JIS X 0212」と「JIS X 0221」も日本工業規格が定めたJIS漢字コードですが, 通常, 「JISコード」に言及する場合, JIS X 0208またはJIS X 0213のどちらかを指します。JIS X 0212とJIS X 0221は, もっとも広く普及した日本語のエンコーディングである「Shift_JIS」に対応していない（前者はISO-2022-JP・EUC-JP・Unicodeで対応, 後者はUnicodeで対応）ことがおもな理由のようです。

ISO-2022-JP

ISO-2022-JPは, 「JIS X 0208」の文字セットを7ビットのコードで表現できるエンコーディング方式です。JISコードに基づいた符号化ということであれば, Unicode, EUC-JP, Shift_JIS, ISO-2022-JPはどれもJISコードに基づいていますが, 通常, 「JISコード」という場合, ISO-2022-JPエンコーディングが示唆されています。

Shift_JIS

ASCII文字コードを8ビットに拡張し, バックスラッシュ→円記号, チルダ→オーバーラインに置き換えて0x80以降のコード範囲に半角カタカナを定義した文字コードは, JIS8コード（8ビットJIS, JISカタカナ）と呼ばれていました。Shift_JISエンコーディングは, この文字コードをさらに拡張して漢字やひらがなが扱えるようにしたものです。通常, こちらを指して「JISコード」と呼ぶことはありません。

エスケープシーケンス（ISO-2022-JP）

ISO-2022では, エスケープシーケンス（切り替え信号）により出力中に文字セットを切り替えます。ISO-2022-JPでは, ASCII・ローマ字（バックスラッシュとオーバーラインが置き換えられた日本版のASCII）・漢字（旧JIS）・漢字（新JIS）のエスケープシーケンスを実装することにより, 英数字と日本語の混合テキストに対応しています。半角カタカナには対応していません。

エスケープシーケンス（ISO-2022-JP-1）

ISO-2022-JPのエスケープシーケンス4種類に加え, 補助漢字（JIS X 0212）の切り替え信号を実装することにより, 扱える文字セットが拡張されたエンコーディングです。JIS X 0212の補助漢字を使用していない場合, 実質的にはISO-2022-JPと同じデータになります。半角カタカナには対応していません。

エスケープシーケンス（ISO-2022-JP-2）

全部で9種類のエスケープシーケンスを実装し, 日本語以外にも簡体字中国語, 韓国朝鮮語。ギリシャ語, 西ヨーロッパ語などの切り替えに対応しています。追加文字セットを使用していない場合, ISO-2022-JP-1と同じデータになります。半角カタカナには対応していません。

半角カタカナ

上述したように, ISO-2022-JPの拡張は, それぞれ補助漢字と日本語以外の言語に対応したものであり, もっとも強力なISO-2022-JP-2であっても, 半角カタカナには対応していません。とはいえ, 実際にはISO-2022-JPに半角カタカナを含めるための方法がいくつか存在します。いずれもISO-2022-JPの非公式な拡張であり, 本来のISO-2022-JPとは区別がつかないため, 相互に合意がないアプリケーション間では情報の交換に失敗します。

JIS7方式（エスケープシーケンス）

半角カタカナの開始を知らせるエスケープシーケンス 1B 28 49 を使用します。

JIS7方式（シフトアウト・シフトイン）

半角カタカナの開始を知らせるコード 0E および終了を知らせるコード 0F を使用します。

JIS8方式

半角カタカナは8ビットコード（A1〜DF）を使用します。

処理系別の対応状況について

下記はWindows 7およびMac OS X 10.6 Snow Leopardの検証結果です。

Windows

コードページ 50220：半角カナは変換に失敗します。

コードページ 50221：半角カナはJIS7方式（エスケープシーケンス）で処理します。

コードページ 50222：半角カナはJIS7方式（シフトアウト・シフトイン）で処理します。

基本となるエンコーディングはISO-2022-JPです。

補助漢字（ISO-2022-JP-1）, 多言語（ISO-2022-JP-2）には対応していません。

Mac OS X

NSISO2022JPStringEncoding：半角カナはJIS7方式（エスケープシーケンス）で処理します。

基本となるエンコーディングはISO-2022-JPです。

補助漢字（ISO-2022-JP-1）, 多言語（ISO-2022-JP-2）には対応していません。

（参考）

Mac OS X (iconv)

ISO-2022-JP：半角カナは変換に失敗します。

ISO-2022-JP-1：半角カナは変換に失敗します。補助漢字は対応しています。

ISO-2022-JP-2：半角カナはJIS7方式（エスケープシーケンス）で処理します。補助漢字, 多言語に対応しています。

概要

Mac OS X/Windowsともに, 基本となるのはISO-2022-JPのようです。Mac OS Xは, ISO-2022-JP-1やISO-2022-JP-2もサポートしています。

ISO-2022-JPの半角カナは, 特に指定しなかった場合, Windowsは非対応ですが, Mac OS Xは対応するようです。対応する場合, どちらもJIS7方式（エスケープシーケンス）で対応します。Windowsは, JIS7方式（シフトアウト・シフトイン）もサポートしてます。JIS8方式は, どちらも対応していません。
